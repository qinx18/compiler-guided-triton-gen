## WAR (Write-After-Read) Dependencies

**Note**: This kernel has WAR (Write-After-Read) dependencies.
If you split the computation into **separate Triton kernels** launched sequentially, kernel launch barriers handle these dependencies naturally — no cloning needed.
Cloning is only needed if reads and writes to the same array happen **within a single kernel**.

**Loop variables** (outer to inner): t, i, j

**Array `p`**: WAR carried by loop(s) `i, j`

**Array `u`**: WAR carried by loop(s) `i, j`
